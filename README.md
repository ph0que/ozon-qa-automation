# Ozon SEO State Auditor (QA Automation) 🚀

### О проекте
Автоматизированный тест на **Python + Pytest** для проверки целостности SEO-данных (Title) на страницах маркетплейса Ozon. 

### В чем "фишка" решения?
В отличие от стандартных тестов, которые просто ищут текст в DOM-дереве, этот инструмент обращается напрямую к объекту состояния фронтенда - `window.__NUXT__`. 

**Преимущества такого подхода:**
1. **Надежность**: Мы проверяем данные в "первоисточнике" (State), что исключает ложные срабатывания при задержках рендеринга или изменениях в верстке.
2. **Скорость**: Извлечение данных через JS-скрипт происходит мгновенно после загрузки объекта.
3. **Обход защиты (Stealth)**: В проекте реализована маскировка через `undetected-chromedriver`, что позволяет обходить системы защиты (WAF), выдающие ошибку 403 (Access Denied) при обычном использовании Selenium.

### Технологический стек
* **Язык**: Python 3.12+
* **Фреймворк**: Pytest
* **Инструменты**: undetected-chromedriver (Selenium-based), JavaScript (для инъекций в стейт).

### 🛠 Инженерные фишки проекта (Почему это надежно)

1. **Bypass WAF & Bot-Detection**: 
   Вместо стандартного Selenium использован `undetected-chromedriver`. Это патчит бинарный файл драйвера на лету, удаляя специфические сигнатуры автоматизации, что позволяет обходить защиту Ozon (Access Denied / 403).

2. **Zero-UI Testing (State Injection)**:
   Тест не тратит время на ожидание отрисовки тяжелых DOM-элементов. Мы делаем инъекцию JS прямо в шину данных фронтенда. Если `window.__NUXT__` пришел с сервера битым - мы узнаем об этом до того, как страница попытается его отрендерить.

3. **Error Persistence**:
   В случае падения теста система автоматически:
   * Делает отладочный скриншот `debug_final.png`.
   * Дампит текущий `page_source.html` для анализа блокировок.
   Это критически важно для отладки тестов в CI/CD пайплайнах.

### 📈 Что можно докрутить (Roadmap)
* **API Interception**: Добавить перехват сетевых запросов (`devtools_version`), чтобы проверять не только стейт, но и ответы от бэкенда на Go.
* **Headless Mode**: Настроить запуск в скрытом режиме для экономии ресурсов сервера при массовом прогоне тестов.
* **Allure Reporting**: Интегрировать красивые отчеты с шагами и графиками.

### 💰 Почему это важно для Ozon?
* **SEO-мониторинг**: Если `window.__NUXT__` придет без Title, поисковые роботы (Google/Yandex) не проиндексируют страницу товара. Это прямая потеря органического трафика и миллионов выручки.
* **Предотвращение регресса**: Автотест ловит ошибки SSR (Server Side Rendering) на ранних этапах, когда визуально страница может казаться целой, но данные для SEO уже «отвалились».
* **Экономия времени**: Благодаря State-тестированию, время прохождения критических сценариев сокращается в 2-3 раза по сравнению с классическим UI-тестированием.
